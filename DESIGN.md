# DESIGN.md — 机加工 MES 生产管控台

## 气质与意象

**想象场景**：上午 9 点，长沙大满制罐厂的生产调度室。阳光透过落地窗洒进来，白色的办公桌上摆放着一杯刚泡好的茶。调度员正在MES系统前查看今日的生产计划，屏幕上白色的背景配合清晰的数据表格，工单状态一目了然。旁边站着车间主任，指着屏幕上那个绿色的"98.5%"合格率数字，满意地点了点头。

**气质关键词**：专业 · 清晰 · 高效 · 简洁 · 易读 · 办公友好

**拒绝的"AI 默认审美"**：
- ❌ 科技蓝 + 圆角卡片 + 渐变光晕
- ❌ 大留白 + 插画风图标
- ❌ 动画炫技、emoji 装饰
- ❌ 过度装饰、花哨配色

## 视觉策略

**默认浅色模式**，白色背景。理由：办公场景下白色背景更清晰易读；打印输出友好；符合企业办公系统惯例；减少长时间使用的眼睛疲劳。

**图形语言**：
- 几何感强，少圆角（4px 极限），多用直角分割线
- 边框 1px 描边代替阴影（`border-border`）
- 数据点用色块（4×4 ~ 6×6 实心方块）而非圆点
- 数字加 `tabular-nums` 等宽对齐，关键 KPI 用大字号
- ID/编码统一等宽字体（`font-mono`）
- 图标用 `lucide-react`，描边 1.5px，颜色和文字同色阶，不抢戏

## 配色方案

### 浅色主题（默认）

```css
/* 背景层级 */
--background: #FFFFFF   /* 页面底色，纯白 */
--card: #F9FAFB         /* 卡片/容器，浅灰 */
--muted: #F3F4F6        /* 输入框/次级容器 */

/* 文字层级 */
--foreground: #1F2937   /* 主要文字，深灰 */
--muted-foreground: #6B7280  /* 次要文字 */
--secondary-foreground: #374151  /* 强调文字 */

/* 描边与分隔 */
--border: #E5E7EB       /* 边框，浅灰 */
--input: #E5E7EB        /* 输入框边框 */

/* 状态色（工业标准） */
--ok:    #22C55E        /* 运行中 / 合格 — 工业绿 */
--warn:  #EAB308        /* 待机 / 让步 — 工业黄 */
--err:   #EF4444        /* 故障 / 不合格 — 工业红 */
--info:  #3B82F6        /* 计划中 / 中性信息 — 工业蓝 */
--accent:#06B6D4        /* 强调色（数据高亮）— 工业青 */

--brand: #F97316        /* 橙色 — 长沙大满品牌色 */

```

### 深色主题（可选，车间大屏场景）

```css
/* 背景层级（由深到浅） */
--bg-0:  #0A0E1A        /* 页面底色，几乎纯黑带蓝 */
--bg-1:  #111726        /* 卡片/容器 */
--bg-2:  #1A2030        /* 浮起/激活态 */
--bg-3:  #242B3D        /* hover */

/* 文字层级 */
--fg-0:  #E8ECF4        /* 主要文字 */
--fg-1:  #A0AABE        /* 次要文字 */
--fg-2:  #6B7592        /* 辅助/标签 */

/* 描边与分隔 */
--line:  #1F2738
--line-strong: #2D3548

/* 状态色（同浅色） */
--ok:    #22C55E
--warn:  #EAB308
--err:   #EF4444
--info:  #3B82F6
--accent:#06B6D4
--brand: #F97316
```

**色彩使用规范**：
- 状态色只用 4 个语义，不要再多
- 大面积用背景色，状态色仅用于关键状态点和数据
- 不在背景上做彩色渐变
- 警示色（红/黄）出现频率受控：1 屏内红色出现 ≤ 3 次，避免视觉疲劳

## 字体排版

```css
--font-sans: "Inter", "PingFang SC", system-ui, sans-serif;
--font-mono: "JetBrains Mono", "SF Mono", "Consolas", monospace;
```

- 数字、ID、编码、状态：等宽（`font-mono`）+ `tabular-nums`
- 中文：PingFang SC
- 西文：Inter
- 字号梯度（少而精）：
  - `text-xs` (12px) — 标签、辅助信息
  - `text-sm` (14px) — 默认正文
  - `text-base` (16px) — 强调文字
  - `text-2xl` (24px) — 卡片标题
  - `text-4xl` (36px) — KPI 数值
  - `text-6xl` (60px) — 大屏核心数据
- 数字一律右对齐
- 中文行高 `leading-relaxed`，英文 `leading-normal`

## 动效与交互

**整体克制**：动效只服务于"让数据看起来在动"，不为了动而动。

- 状态变化：颜色切换 + 微小背景闪烁（200ms），不做位移
- 数字更新：滚动数字（count-up），1.5s 缓动
- 实时数据刷新：呼吸式指示点（小绿点 1.5s 缓慢脉动）
- Hover：边框颜色 + 0.2s 过渡
- 缓动函数：`cubic-bezier(0.4, 0, 0.2, 1)` (Material standard)
- 不用 bounce，不用 spring

## 组件规范

**KPI 卡片**：
```
┌──────────────────────┐
│ 今日产量    ▲ 12%    │  ← 标签 + 趋势
│                       │
│    1,247 / 1,400      │  ← 大数字 + 目标
│ ███████████░░  89%    │  ← 进度条
└──────────────────────┘
```

**状态标签**：
- 浅色主题：用 `bg-{color}/10 text-{color} border-{color}/30` 三件套
- 不用纯色块（避免刺眼）
- 圆角 2px，配 1px 描边

**表格**：
- 表头 muted 加粗
- 行高 44px（不要紧凑的 36）
- 状态列用色块 + 文字
- 数字列右对齐
- Hover 高亮 accent

**按钮**：
- 主要：橙色 `bg-primary` 仅用于关键操作（创建工单、报工等）
- 次要：透明 + 边框
- 危险：红色描边
- 不用阴影

## 设计禁忌

- ❌ 不用 emoji
- ❌ 不用渐变背景
- ❌ 不用大阴影（最多 `shadow-sm`）
- ❌ 不用 bounce 动画
- ❌ 不用圆角超过 4px 的容器
- ❌ 不用彩色插画图
- ❌ 不用超过 2 种装饰色同时出现
- ❌ 文字不用大写 letter-spacing（破坏中文阅读）
- ❌ 不做"AI 助手"、"智能推荐"这种话术

## 信息密度

- 一屏 1 看板页至少能呈现：4 个 KPI + 1 个趋势图 + 1 个设备状态矩阵 + 1 个工单进度列表
- 表格 1 屏至少 10 行
- 不留白过度，能并排就并排，能折叠就折叠

## 主题切换

系统支持浅色/深色主题切换：
- `mes-light`：白色背景（默认）
- `mes-dark`：工业橙深色
- `mes-cyan`：冰蓝深色
- `mes-emerald`：翡翠绿深色
- `mes-crimson`：朱红深色
- `mes-amber`：琥珀金深色

切换方式：`html[data-theme="xxx"]`
